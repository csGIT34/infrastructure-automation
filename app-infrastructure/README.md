# app-infrastructure

GitOps repository for infrastructure deployments. Managed by the infrastructure MCP server.

## Branch Model

- `main` - Workflows and shared config only
- `{app}/{env}` branches (e.g., `myapp/dev`, `myapp/prod`) - Application infrastructure

## How It Works

1. Developer requests infrastructure via Claude Code + MCP server
2. MCP server calls `push_tfvars` which creates/updates a branch
3. Push triggers the `terraform-apply.yaml` workflow
4. Terraform applies the pattern with the provided tfvars

## Branch Structure

Each app/env branch contains pattern directories:

```
key_vault/
  terraform.tfvars.json    # Generated by MCP server
  backend.hcl              # Terraform state config
postgresql/
  terraform.tfvars.json
  backend.hcl
```

## State Management

Terraform state is stored in Azure Storage:
- Key format: `{business_unit}/{env}/{project}/{pattern}-{name}/terraform.tfstate`
- Configured via `backend.hcl` in each pattern directory
